\section{Bytecode Interpreters} 

  Java and Python run on a bytecode VM. The process is that after you scan, you take the list of tokens and convert them into bytecode. 

  \begin{definition}[Bytecode]
    \textbf{Bytecode} is a low-level compact instruction sequence---where each instruction/\textbf{opcode} is usually encoded in a byte---produced from source code and consumed by a virtual machine or interpreter. 
  \end{definition} 

  The fact that each instruction is a byte limits us to a total of 256 instructions. Usually, this is enough but sometimes, we may need to apply variable-length encoding to instructions. 

  \begin{definition}[Virtual Machine]
    A \textbf{virtual machine} is a tuple $(T, \texttt{ip})$ that contains 
    \begin{enumerate}
      \item a pointer to the sequence of tokens generated by the scanner. 
      \item an \textbf{instruction pointer} \texttt{ip} that points to the next opcode.\footnote{Note that this instruction pointer is specific to the VM, not the actual CPU.}
    \end{enumerate}
  \end{definition}

  There are two main types of virtual machines: stack-based and register-based. These are based off of the execution model, not whether the final machine code uses stacks or registers. 

\subsection{Stack-Based Virtual Machines} 

  The key to stack-based virtual machines is the realization that \hyperref[dsa-thm:tree_traversal_dfs]{traversing a tree (AST) is really just the same thing as DFS, which uses a stack}. Therefore, all the evaluations and executions can be done with a stack. 

  \begin{definition}[Stack-Based Virtual Machine]
    A \textbf{stack-based virtual machine} is a virtual machine $(T, \texttt{ip}, S)$ that contains an extra \hyperref[dsa-def:stack]{stack} $S$. 

    \begin{figure}[H]
      \centering 
      \begin{lstlisting}
	#define STACK_MAX 256

	typedef struct {
	  Chunk* chunk;
	  uint8_t* ip;
	  Value stack[STACK_MAX]; 
	  Value* stackTop;
	} VM;
      \end{lstlisting}
      \caption{C implementation of a stack-based virtual machine.} 
      \label{fig:stack_based_vm}
    \end{figure}
  \end{definition}

  This allows us to define the relevant opcodes. 

  \begin{definition}[\texttt{OP\_CONSTANT}]
    The \texttt{OP\_CONSTANT} is a bytecode instruction that tells the VM to load a constant value from the chunk's constant pool and push it on to the VM stack. 
  \end{definition}

  \begin{definition}[Arithmetic]
    The arithmetic operators all bytecode instructions that tell the VM to do the same thing. 
    \begin{enumerate}
      \item \texttt{OP\_ADD} says to pop 2 values, add them, and push the result. 
      \item \texttt{OP\_SUBTRACT} says to pop 2 values, subtract them, and push the result. 
      \item \texttt{OP\_MULTIPLY} says to pop 2 values, multiply them, and push the result. 
      \item \texttt{OP\_DIVIDE} says to pop 2 values, divide them, and push the result. 
    \end{enumerate}
  \end{definition} 

  \begin{definition}[Negation]
    The \texttt{OP\_NEGATE} is a bytecode instruction that tells the VM to pop 1 value, negate it, and push the result to to stack. 
  \end{definition}

  It is often nice to see the state of the stack within each instruction. This is called the \textbf{stack trace}. 

  \begin{example}[Adding 2 Numbers]
    \begin{lstlisting}
      Instruction      Stack

      OP_CONSTANT 0    [1]
      OP_CONSTANT 1    [1, 2]
      OP_ADD           [3]
    \end{lstlisting}
  \end{example}

  \begin{definition}[\texttt{OP\_RETURN}]
    The \texttt{OP\_RETURN} is a bytecode instruction that tells the VM to return the current value(?)
  \end{definition}

\subsection{Register-Based Virtual Machines}

  \begin{definition}[Register-Based Virtual Machine]
    A \textbf{register-based virtual machine} is a virtual machine $(T, \texttt{ip}, R)$ that contains an extra set of \textit{registers}. 

    \begin{figure}[H]
      \centering 
      \begin{lstlisting}
	#define REGISTER_MAX 256

	typedef struct {
	  Chunk* chunk;
	  uint8_t* ip;
	  Value registers[REGISTER_MAX];
	} VM;
      \end{lstlisting}
      \caption{C implementation of a register-based virtual machine.} 
      \label{fig:register_based_vm}
    \end{figure}
  \end{definition}

  \begin{example}[Adding 2 Numbers]
    \begin{lstlisting}
      LOAD_CONST r0, 0
      LOAD_CONST r1, 1
      ADD r2, r0, r1
      RETURN r2 
    \end{lstlisting}
  \end{example}
